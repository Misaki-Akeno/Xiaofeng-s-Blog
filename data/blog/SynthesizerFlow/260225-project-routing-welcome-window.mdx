# 路由重构与全新欢迎窗口：更流畅的用户体验

我们很高兴宣布最新的更新，专注于改善首次用户体验，并将项目状态与应用的 URL 路由深度集成。

## 有什么新功能？

### 🛣️ 基于路径的项目路由

以前，项目使用查询参数加载（例如 `/?project=my-project-id`）。在本次更新中，我们完全重构了 Next.js App Router 设置，使用路径变量（例如 `/[locale]/[projectId]`）。

- **更清晰的 URL**：分享项目现在更加直观，使用 `/zh-CN/my-project` 这样的清晰路径。
- **可分享性**：基于路径的路由是公共项目更好的可索引性和更紧密的平台集成的基础。
- **无缝 URL 同步**：当您从侧边栏加载或切换项目时，URL 会使用 Next.js 的浅路由自动更新，无需完整页面刷新，保留您的音频上下文和状态。

### 👋 全新欢迎窗口

为了提供更好的引导体验，访问应用根路径（例如 `/zh-CN`）的用户现在会看到一个专注的欢迎窗口。

- **快速开始**：直接从中央对话框创建空白项目（"新建项目"）或探索内置模板（"加载示例项目"）。
- **智能缓存与自动加载**：应用会智能检查您的本地持久化状态。如果您的缓存中有活跃的未保存项目，Canvas 会立即自动加载，抑制欢迎窗口以确保您的创作流不被中断。
- **自动预取**：当欢迎窗口打开时，示例项目会在后台静默预取。点击"加载示例项目"会立即打开完全加载到内置预设的项目管理器侧边面板。

## 未来展望

这种 UX 优化确保了无论您是直接回到上次本地草稿的高级用户，还是寻找复音合成器模板的新手，SynthesizerFlow 都能安全优雅地处理状态。
